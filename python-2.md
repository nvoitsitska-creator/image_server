### **1. Зберігання метаданих про зображення в базі даних**

**Опис:**

  - унікальний ідентифікатор зображення;

  CREATE TABLE images (

      file_type TEXT NOT NULL

відповідні метадані повинні бути додані до бази даних;

### **4. Пагінація списку зображень**

Нагадаємо, пагінація — це розділення великого обсягу інформації на окремі сторінки. 
Уявіть книгу — було б незручно читати її як один довгий сувій. 
Те саме і з даними на веб-сторінці. Якщо у нас 1000 зображень, показувати їх всі одразу буде складно. 
Це важко для браузера, буде довго завантажуватися, ну а користувачеві буде просто незручно читати. 

Тому ми розбиваємо список на сторінки, наприклад по 10 елементів.

**Опис:**

- якщо список зображень задовгий, його потрібно розділити на сторінки.

- реалізувати пагінацію на сторінці `/images-list`:

  - відображати по 10 зображень на одній сторінці.
  - додати кнопки "Наступна сторінка" та "Попередня сторінка".

**Вимоги**

- Використовувати `LIMIT` та `OFFSET` в SQL-запитах:

  ```sql
  SELECT * FROM images ORDER BY upload_time DESC LIMIT 10 OFFSET <зміщення>;
  ```

- На веб-сторінці додати навігацію для переходу між сторінками:

  - наприклад, `/images-list?page=2`.

- Переконатися, що пагінація коректно працює при малій кількості зображень або досягненні кінця списку.

### Створення таблиці `images` для зберігання метаданих

**Створення таблиці `images` в PostgreSQL:**

Таблиця повинна бути створена з використанням наступної структури:

```sql
CREATE TABLE images (
    id SERIAL PRIMARY KEY,               -- Унікальний ідентифікатор запису
    filename TEXT NOT NULL,              -- Унікальна назва файлу (згенерована)
    original_name TEXT NOT NULL,         -- Оригінальна назва файлу (користувача)
    size INTEGER NOT NULL,               -- Розмір файлу в байтах
    upload_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP, -- Час завантаження файлу
    file_type TEXT NOT NULL              -- Формат файлу (jpg, png, gif тощо)
);
```

Поля таблиці:

- **`id`**: автоматично генерований унікальний ідентифікатор для кожного зображення.
- **`filename`**: унікальна назва файлу, згенерована сервером.
- **`original_name`**: назва файлу, надіслана користувачем.
- **`size`**: розмір файлу в байтах.
- **`upload_time`**: дата та час завантаження файлу.
- **`file_type`**: формат файлу (наприклад, `jpg`, `png`).

### Підключення Python-бекенду до бази даних

**Підключення до бази даних PostgreSQL:**

- використовуйте бібліотеку `psycopg2` для взаємодії з PostgreSQL.

- налаштуйте підключення через параметри:

  - хост (наприклад, `db`, якщо використовується Docker Compose).
  - порт (за замовчуванням `5432`).
  - назва бази даних, користувач та пароль.

Приклад підключення:

```python
import psycopg2

conn = psycopg2.connect(
    dbname="images_db",
    user="postgres",
    password="password",
    host="db",
    port="5432"
)
cursor = conn.cursor()
```

### Додавання даних в PostgreSQL

**Додавання запису в базу даних:**

Реалізувати функцію для збереження метаданих:

```python
def save_metadata(filename, original_name, size, file_type):
    query = """
    INSERT INTO images (filename, original_name, size, file_type)
    VALUES (%s, %s, %s, %s)
    """
    cursor.execute(query, (filename, original_name, size, file_type))
    conn.commit()
```
Ця функція викликається після успішного збереження файлу на сервері.
